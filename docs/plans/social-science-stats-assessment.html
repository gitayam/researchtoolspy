Section B

Tooling verdict — in-browser statistics library comparison for this stack
Bottom line: Use simple-statistics + mljs ecosystem (ml-pca, ml-regression, ml-matrix) + label-score/krippendorff as a composed in-browser stack. This covers Tiers A and B without any WASM overhead. Reserve WebR for a dedicated "Advanced Analysis" escape hatch rendered in an iframe/panel — do NOT run it in the Worker. Pyodide is not viable here.

Section C

Phased proposal — MVP module and stretch capabilities

Phase 1 — MVP: "Quantitative Analysis" module

Estimated: 3–5 weeks · Stack: TS + simple-statistics + ml-pca + label-score · No new backend services
  • Dataset entry point: new tab/panel within existing DatasetPage or separate route /analysis; accepts any tabular dataset (CSV upload or select from existing Dataset objects). Parse columns, infer types (numeric vs categorical).
  • Descriptives tab: per-column mean, median, SD, IQR, min/max, skewness, frequency table for categoricals. Render using existing chart components (bar + box-plot approximation).
  • Correlation tab: select any N numeric columns → Pearson matrix heat-map + Spearman toggle. Highlight statistically significant pairs (t-distribution p-value via jStat).
  • Cross-tab / chi-square tab: pick two categorical columns → contingency table, chi-square test, Cramér's V effect size, interpret automatically ("weak / moderate / strong").
  • Group comparison tab: pick a categorical grouping + numeric variable → Welch t-test (2 groups) or one-way ANOVA (3+) with p-value and effect size (Cohen's d / eta-squared).
  • Simple regression tab: X and Y picker → scatter + fitted line, slope, R², p-value, residual plot.
  • Export: analysis result as JSON persisted to D1 (see data model below) + "Download R script" button that generates a pre-populated R template (extends existing R-script pattern).

Phase 2 — Reliability Engine (highest platform-unique value)

Estimated: 2 weeks · integrates with COP evidence coding and claims workflows
  • Unique differentiator: The platform codes claims, evidence, and SATS evaluations using multiple analysts. Inter-rater reliability is the methodological backbone for any claim that "coders agreed."
  • IRR upload: CSV with columns [rater, item, code] → compute Cohen's kappa (2 raters), Fleiss' kappa (3+), or Krippendorff's alpha (any scale type + missing data) via label-score + krippendorff npm packages.
  • Interpretation layer: label-score already includes Landis-Koch thresholds (kappa <0.2 = poor … >0.8 = almost perfect). Surface these in plain language for non-statistician analysts.
  • SATS integration: run Cronbach's alpha across the 8 SATS dimensions (reliability/credibility/validity…) for a set of evaluations to measure evaluator consistency — directly actionable for calibration.
  • COP link: offer IRR calculation on COP session claim codes — "how much do your analysts agree on whether each claim is verified/unverified/deceptive?"

Phase 3 — Multivariate & Regression

Estimated: 2–3 weeks · adds mljs packages
  • Multiple OLS: ml-regression-multivariate-linear; coefficient table with p-values (jStat t-distribution); VIF for multicollinearity; standardised betas.
  • Logistic regression: ml-regression logistic; predict binary outcome (e.g. claim verified = 1); ROC/AUC visualization.
  • PCA: ml-pca; scree plot, biplot, component loadings table. Label each component via AI (OpenAI call: "given these loadings, suggest a name for this component").

Phase 4 (Stretch) — WebR Integration

Estimated: 3–4 weeks · browser-side only, never in Worker
  • Add a "Run in R" toggle on any analysis result: loads WebR runtime in a browser-side Web Worker (not the CF Worker), executes the generated R script against the uploaded data, returns full output.
  • Unlocks: EFA (psych::fa), CFA (lavaan), multilevel models (lme4), survey weighting — anything in the ~10,500 WASM-ready CRAN packages.
  • Implementation pattern: R script template already exists in docs/r-scripts/; WebR renders output (tables, plots as PNG) back into the page via postMessage.
  • Caveat: ~25–35 MB initial download; show loading indicator; gate behind a "Advanced / R analysis" feature flag.
  • Do NOT expand the existing R-script export templates with SEM/CFA until Phase 4 confirms user demand — over-engineering risk is real.
